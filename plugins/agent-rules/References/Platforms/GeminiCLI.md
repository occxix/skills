# Gemini CLI Project Instructions

Use `GEMINI.md` for project-level Gemini instructions.

## Priority

1. Project `GEMINI.md`
2. Parent-directory `GEMINI.md` when the repo intentionally inherits from it
3. `~/.gemini/GEMINI.md` for global defaults

## Writing rule

- Keep the file focused on project conventions, commands, and guardrails.
- Use `@file` references for longer rule sets when needed.
