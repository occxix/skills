# Notes and Knowledge Index

Use these rules for Obsidian vaults, Markdown notes, and life-organization
folders.

## Knowledge

- `Knowledge/Obsidian.md`
- `Knowledge/Markdown.md`

## Life Support

- `Life/PersonalOps.md`

## Core Constraints

- Keep notes searchable, linkable, and easy to archive.
- Use stable heading structure and predictable filenames.
- Avoid deep folder nesting unless the information model truly needs it.
- Separate active work from archive and reference material.
