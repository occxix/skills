# Requirement Analysis Rules

Use this file when turning a user request into a scoped project or feature plan.

## Core Rules

- Clarify the goal, constraints, success criteria, and non-goals.
- Identify the smallest useful scope that solves the request.
- Surface ambiguities early instead of guessing.
- Split large requests into smaller subproblems when needed.
- Map the request to existing architecture and project conventions.

## Output

- A concise problem statement
- Scope boundaries
- Assumptions and open questions
- Recommended next step
