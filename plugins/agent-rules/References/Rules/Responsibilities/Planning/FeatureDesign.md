# Feature Design Rules

Use this file when designing how a feature should work before implementation.

## Core Rules

- Define inputs, outputs, workflow, and boundary conditions.
- Identify domain concepts and ownership boundaries.
- Keep the design aligned with the current architecture.
- Prefer reusable feature slices over one-off special cases.
- Include validation, error handling, and success paths in the design.

## Output

- Feature flow
- Data model or contract sketch
- Edge cases
- Acceptance conditions
