# UI Design Rules

Use this file when designing screens, layouts, interaction flow, or visual hierarchy.

## Core Rules

- Define the primary user action first.
- Keep the layout clear, focused, and consistent with the product style.
- Separate information hierarchy from visual decoration.
- Make empty states, loading, validation, and error states explicit.
- Prefer a small number of strong interactions over many weak interactions.

## Output

- Screen structure
- State map
- Interaction notes
- Visual direction
