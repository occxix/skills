# Feature Acceptance Rules

Use this file when verifying that a feature is ready to ship or hand over.

## Core Rules

- Validate the feature against the agreed scope and success criteria.
- Check the user-visible behavior, not just internal implementation.
- Confirm that critical paths, error paths, and edge cases work.
- Note remaining gaps clearly if anything is out of scope.
- Keep the acceptance report short and factual.

## Output

- Pass/fail summary
- Confirmed behavior
- Remaining issues
- Recommended next action
