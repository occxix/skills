# Integration Test Rules

Use this file when validating behavior across components, services, or external adapters.

## Core Rules

- Test real collaboration points and integration boundaries.
- Keep setup realistic but controlled.
- Verify data flow, contract compatibility, and wiring.
- Avoid over-mocking the system under test.
- Make test fixtures explicit and reusable.

## Output

- End-to-end component interactions
- Boundary verification
- Environment/setup notes when needed
