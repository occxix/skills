# Unit Test Rules

Use this file when writing tests for isolated functions, classes, or modules.

## Core Rules

- Test one behavior per test when possible.
- Use clear arrange-act-assert structure.
- Prefer deterministic tests with minimal external dependencies.
- Cover success, failure, and edge cases.
- Keep test names descriptive and behavior-focused.

## Output

- Focused test cases
- Clear failure messages
- Minimal mock setup
