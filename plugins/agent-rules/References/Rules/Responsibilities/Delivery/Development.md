# Development Rules

Use this file when implementing business functionality or application behavior.

## Core Rules

- Follow the repo’s architecture and existing conventions.
- Keep changes cohesive within the feature or layer.
- Prefer adding new code over rewriting stable code.
- Keep dependencies explicit and testable.
- Preserve existing public contracts unless the user asks to change them.

## Output

- Implementations that match the planned design
- Minimal, focused changes
- Clear wiring or registration where needed
