# Go Rules

Use this file for Go projects.

## Core Rules

- Keep packages small and cohesive.
- Prefer simple interfaces and explicit dependencies.
- Use context-aware APIs for I/O and long-running work.
- Keep error handling explicit and local.
- Follow the repository’s linting and formatting conventions.

## Avoid

- Overengineering with unnecessary abstraction.
- Global mutable state unless the project already relies on it.
