# Java Rules

Use this file for Java projects.

## Core Rules

- Keep packages aligned with the architecture.
- Prefer clear service boundaries and dependency injection when the project uses it.
- Use checked/unchecked exceptions consistently with the repo’s conventions.
- Keep public APIs stable unless the user asks for a contract change.
- Follow the project’s formatter and static analysis rules.

## Avoid

- Large utility classes with mixed responsibilities.
- Magic constants and hidden coupling.
