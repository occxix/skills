# 小说章节解析器 (novel-chapter-parser)

一个 Claude 插件，用于自动解析小说章节内容，构建结构化的知识库，包含人物、组织、地域、剧情、关系网络等信息。

## ✨ 功能特性

### 🚀 核心功能
- **三次操作流程**：严格按照「纯提取 → 仅查询 → 全更新」的流程处理，避免误改已有信息
- **自动内容提取**：自动识别章节中的人物、组织、地域、事件、对话等关键信息
- **多格式支持**：支持单文件（整本书）和多文件（按章分文件）两种小说存储格式
- **批量处理**：支持一次性处理多个章节，提高效率
- **智能编码识别**：自动识别 UTF-8、GBK、GB18030、Big5 等常见中文编码，自动修复乱码

### 📚 知识库结构
自动生成完整的知识库目录结构：
```
小说解析器知识库（《小说名称》）
├── 0_预处理/              # 章节预处理和查询报告
├── 1_写作手法/            # 写作手法分析和汇总
├── 2_世界观与环境/
│   ├── 地域库/            # 地理、场所、地点实体
│   ├── 组织库/            # 门派、势力、机构等组织实体
│   └── 世界观总览.md       # 全局世界观设定
├── 3_人物体系/            # 人物实体，每人一个独立文档
├── 4_剧情与事件/          # 主线、支线、关键事件记录
├── 5_关系网络/            # 人物、组织关系图谱和变更记录
├── 6_动态演变与主题/      # 主题分析、剧情走向深度分析
├── 7_变化日志/            # 各类变更记录和全局时间线
├── 8_章节总结/            # 单章总结文件
└── 9_关键记忆点/          # 高能语录、思想变化、剧情转折记录
```

## 📦 安装方法

### 方法 1：通过插件市场安装
```
/plugin marketplace add csvkse/skills
```

### 方法 2：手动安装
1. 克隆仓库到本地
2. 在 Claude 中执行 `/plugin install /path/to/novel-chapter-parser`
3. 执行 `/plugin enable novel-chapter-parser` 启用插件

## 🎯 使用指南

### 1. 初始化知识库
首先为要阅读的小说创建知识库结构：
```bash
/init-novel <小说名称> [知识库路径]
```

示例：
```bash
/init-novel 诡秘之主
```
会在当前目录下创建 `小说解析器知识库（《诡秘之主》）` 目录结构。

### 2. 处理单章
处理指定章节的内容：
```bash
/parse-chapter <章节号> <小说原文路径> [知识库路径]
```

示例：
```bash
/parse-chapter 1 ./诡秘之主.txt
```

执行后会生成：
- `0_预处理/第1章预处理.md`：自动提取的章节内容
- `0_预处理/第1章查询报告.md`：知识库查询结果和更新建议

### 3. 批量处理章节
一次性处理多个连续章节：
```bash
/parse-chapters <起始章> <结束章> <小说原文路径> [知识库路径]
```

示例：
```bash
/parse-chapters 1 10 ./诡秘之主.txt
```

### 4. 完成知识库更新
根据预处理和查询报告的结果，手动（或自动）完成知识库的更新：
1. 创建/更新实体文件（人物、组织、地域等）
2. 追加变化日志记录
3. 生成章节总结
4. 记录关键记忆点

## ⚙️ 配置选项

可以通过 Claude 设置页面配置以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `default_knowledge_base_path` | string | `./novel-knowledge-base` | 默认知识库存储路径 |
| `auto_extract_enabled` | boolean | `true` | 是否启用自动内容提取功能 |
| `auto_confirm_updates` | boolean | `false` | 是否自动确认实体更新（不提示用户） |

## 📝 处理流程说明

### 操作 1：预处理（纯提取）
> ✅ 只读本章原文，不查询知识库，不做任何判断
- 自动识别章节标题、字数、时间地点
- 提取出现的人物、组织、地域实体
- 提取关键事件、对话、伏笔
- 生成预处理模板文件

### 操作 2：查询知识库
> ✅ 只查询已有信息，不做任何修改
- 根据预处理结果，查询相关实体的当前状态
- 生成查询报告，列出需要新增或更新的实体
- 判断是否需要参考历史时间线或之前的章节总结

### 操作 3：更新知识库
> ✅ 一次性完成所有更新操作
1. 创建/更新实体文件
2. 追加各类变化日志
3. 更新总时间线
4. 生成章节总结
5. 记录关键记忆点

## 🛠️ 支持的章节标题格式

插件会自动识别以下章节标题格式：
- 中文数字：`第一章`、`第一百二十章`
- 阿拉伯数字：`第1章`、`第120章`
- 英文格式：`Chapter 1`、`Episode 120`
- 其他格式：`卷一`、`第1节`、`第1回`
